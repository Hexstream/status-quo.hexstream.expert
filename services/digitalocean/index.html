⬉ ⚓ ⬇ DigitalOcean » My config » Droplets » spike.hexstream.net » Apache

For installation of Apache, I simply use the apache2 Ubuntu package.

I have this at the end of my /etc/apache2/apache2.conf (because I want to mostly ignore the Ubuntu way of doing things, in this case):

Include custom/custom.conf

The custom directory is a copy of the $PRE_DEPLOY_DIR/apache/ directory, pulled in by deploy.sh. $PRE_DEPLOY_DIR/apache/ is automatically populated with the files in the services/digitalocean/config/droplets/spike.hexstream.net/apache/ directory (recursively) in the status-quo.hexstreamsoft.com repo by pre-deploy.sh whenever the master branch is updated, thanks to a git post-receive hook. Whenever one of the apache config files is updated, I manually invoke the deploy.sh (as root), for now.

$PRE_DEPLOY_DIR is defined in /etc/environment and has the value /usr/local/etc/pre-deploy.

Here's my /etc/apache2/custom.conf.

The aforementioned custom.conf Includes site-wide-redirects.conf and site-wide-tunnels.conf. That latter file's $HOME_IP is replaced by my current home IP address using sed during pre-deployment.

The deploy.sh script restarts the Apache server with apache2ctl graceful.
(graceful is like restart except it will finish serving the current requests instead of abruptly aborting them.)
